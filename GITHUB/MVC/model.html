What is the Model?

The Model represents the application's data structure and core business logic. It is directly responsible for managing the data, rules, and logic of the application. It receives data inputs from the Controller and responds to instructions to update or fetch information, completely independent of how that data will eventually look on the screen.

Responsibilities of the Model

  1. Managing the state of the application.
  2. Enforcing business logic, rules, and data validation.
  3. Interacting with databases (performing CRUD operations: Create, Read, Update, Delete).
  4. Notifying the system or Controller when its data state changes.

Examples of Data Managed by the Model

In a standard web application (like an E-Commerce store), the Model manages structural data fields such as:

Data Asset Data Type / Examples Description
User Credentials Username, Encrypted Password, Email Saves registration details and verifies security log-ins.
Product Inventory Product ID, Price, Stock Quantity, SKU Tracks catalog availability and item attributes.
Order Details Transaction ID, Timestamp, Total Price Logs commercial history and tracks order statuses.
Customer Profile Shipping Address, Phone Number Stores individual client identification details.
Shopping Cart State List of Item IDs, Temporary Session Data Holds active user choices before a final checkout occurs.
