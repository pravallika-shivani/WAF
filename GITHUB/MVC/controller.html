What is the Controller?

The Controller acts as the brains or the intermediary bridge between the Model and the View. It intercepts incoming inputs and requests sent from a user interacting with the View, processes or reformats that input, commands the Model to manipulate information accordingly, and instructs the View to update its presentation layer.

Responsibilities of the Controller

  • Interpreting mouse clicks, keystrokes, and submitted form inputs from the user interface.
  • Routing web requests (handling page links and URLs).
  • Calling upon specific functions within the Model to pull or append raw information.
  • Selecting the correct View layout template to return back to the user's browser.

User Request Processing Workflow

The step-by-step lifecycle of an execution request in MVC occurs in the following procedural order:

  1. User Interaction (View): A customer visits the webpage and fills out an input field (e.g., searches for a book) and hits "Submit".
  2. Intercept Request (Controller): The Controller catches this action and parses the submitted search keywords.
  3. Update/Fetch Data (Model): The Controller commands the Model to query the system for matching items.
  4. Process Result (Model): The Model searches the underlying database, retrieves the raw results, and hands them back to the Controller.
  5. Deliver Response (Controller -> View): The Controller injects this raw data bundle into the designated View template.
  6. Render Screen: The View translates the data into an organized webpage layout for the user to read.
